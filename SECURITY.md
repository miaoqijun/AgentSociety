# Security Policy

## Supported versions

We generally support the latest released versions on PyPI. If you are using an older version, please upgrade first and re-check whether the issue still exists.

## Reporting a vulnerability

Please **do not** open a public issue for suspected security vulnerabilities.

Instead, report privately to the maintainers with:

- A clear description of the issue and impact
- Steps to reproduce (PoC if possible)
- Affected versions / commit hash
- Any suggested fix or mitigation

If you are unsure how to reach the maintainers privately, you may open an issue with minimal details and request a private contact channel.

## Disclosure

We will acknowledge receipt, investigate, and coordinate a fix and release. We aim to provide an initial response within a reasonable timeframe.
