# Interactive EDA Inside Analysis Reports

`run-eda` already produces **interactive HTML** under `presentation/hypothesis_{id}/data/`:

| `analysis_plan.eda_profile` | Typical outputs                                  | Interactivity                                             |
| --------------------------- | ------------------------------------------------ | --------------------------------------------------------- |
| `ydata`                     | `eda_profile.html`, `eda_profile_<table>.html`   | ydata-profiling widgets (expand sections, toggle details) |
| `sweetviz`                  | `eda_sweetviz.html`, `eda_sweetviz_<table>.html` | Sweetviz paired exploration UI                            |
| `quick-stats`               | `eda_quick_stats.md`                             | Static — summarize in prose + table                       |
| `correlation` / `missingno` | PNG/HTML per tool                                | Embed image or link                                       |

The **analysis report HTML** should surface this interactivity, not only link away in appendix.

## Recommended pattern (report HTML)

1. **§数据** still has prose + summary `.data-table` (required for readers who skim).
2. Add **「交互式 EDA」** block with tabs (see `report-shell.reference.html`):
   - Tab **摘要** — same bullets as MD §数据
   - Tab **交互探索** — `<iframe src="data/eda_profile.html">` (or `eda_sweetviz.html`)
3. **Appendix** — deep link list of all `data/eda_*` files from `evidence_index.json`

### iframe snippet (copy & adjust path)

```html
<section class="eda-interactive" id="eda-interactive">
  <h2>数据与探索</h2>
  <div class="tab-bar" role="tablist">
    <button type="button" class="tab active" data-tab="summary">摘要</button>
    <button type="button" class="tab" data-tab="explore">交互式 EDA</button>
  </div>
  <div class="tab-panel active" data-panel="summary">
    <!-- prose + data-table -->
  </div>
  <div class="tab-panel" data-panel="explore">
    <p class="iframe-hint">下方为 run-eda 生成的可交互画像（ydata/Sweetviz）。若空白，请用浏览器直接打开 <a href="data/eda_profile.html">data/eda_profile.html</a>。</p>
    <iframe
      class="eda-frame"
      src="data/eda_profile.html"
      title="Interactive EDA profile"
      loading="lazy"
    ></iframe>
  </div>
</section>
```

Use `eda_sweetviz.html` when plan used Sweetviz. Multi-table runs: iframe `eda_profile.html` index page lists tables (already generated by Python).

## Paths & preview

- iframe `src` is **relative to the report HTML file** (`presentation/hypothesis_{id}/report_zh.html` → `data/eda_profile.html`).
- VS Code **Live Preview** and local `file://` usually resolve sibling `data/` correctly; if iframe is blank, open the EDA file in a new tab (link in hint).
- Do **not** paste ydata/Sweetviz HTML inline — files are large and self-contained.

## Markdown reports

MD cannot iframe reliably in all viewers. Use:

- Link: `[打开交互式 EDA](data/eda_profile.html)`
- One-line instruction: “完整交互画像见 `data/eda_profile.html`”

HTML reports are where interactivity shines.

## Harness / quality

- Register EDA paths with `record-phase-artifacts` after `run-eda`.
- `build-report-context` lists them under section `data`.
- report-reviewer checks that §数据 mentions EDA and HTML iframe targets exist on disk.
