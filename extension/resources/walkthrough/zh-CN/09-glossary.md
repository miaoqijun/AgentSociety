## 术语附录

这里整理 AI Social Scientist 和 AgentSociety² 中常见的关键名词，按中文拼音首字母分组。第一次使用时不需要全部记住；遇到概念不清楚时回来查即可。

快速索引：[A-C](#a-c) · [D-F](#d-f) · [H-J](#h-j) · [L-M](#l-m) · [P-R](#p-r) · [S-T](#s-t) · [V-Z](#v-z)

### A-C

| 索引 | 术语 | 说明 |
|------|------|------|
| A | **Agent 运行时技能** | 安装到 `custom/skills/`，主要供模拟中的 Agent 使用。 |
| A | **AgentSociety²** | LLM 原生的智能体研究框架，用于把社会科学问题组织成可运行、可复查的计算实验。 |
| A | **AGENTS.md** | 面向通用编码助手的项目说明文件，记录项目结构、运行命令和协作约定。 |
| A | **AI Chat** | 插件中的对话入口，可用于询问研究问题、生成配置、检查结果或辅助操作项目。 |
| A | **AI Social Scientist** | VS Code 插件和研究工作台，负责项目管理、配置、技能、回放和帮助入口。 |
| A | **API Base** | 模型服务的访问地址，例如 `https://api.openai.com/v1`。 |
| A | **API Key** | 调用模型服务所需的密钥。请保存在本地配置中，不要提交到公开仓库。 |
| C | **Claude Code 技能** | 安装到 `.claude/skills/`，主要供 Claude Code 等编码助手在 IDE 中使用。 |
| C | **CLAUDE.md** | 面向 Claude Code 的项目说明文件，用于告诉编码助手如何理解和操作当前项目。 |

### D-F

| 索引 | 术语 | 说明 |
|------|------|------|
| D | **大语言模型（LLM）** | 例如 GPT、Claude、DeepSeek、通义千问等，用于理解文本、生成方案和辅助推理。 |
| E | **`.env`** | 当前工作区的本地环境配置文件，常用于保存 API 地址、密钥和模型名称。 |
| E | **Embedding Model** | 用于把文本转换成向量的模型，常用于语义搜索和文献检索。 |
| E | **Environment（智能体环境）** | Agent 所处的模拟环境，可以包含空间、资源、经济规则、社交关系或自定义机制。 |
| F | **FastAPI 后端** | 在本地运行的 Python 服务，负责 LLM 调用、实验执行、技能管理和 API。 |

### H-J

| 索引 | 术语 | 说明 |
|------|------|------|
| H | **后端服务（Backend）** | 插件背后的本地 Python 服务。前端界面负责交互，后端负责模型调用、实验运行和数据服务。 |
| H | **回放数据（Replay）** | 实验过程记录，用于回看 Agent 行为、环境变化和关键结果。 |
| J | **经验范式** | 通过观察、问卷、实验或行为数据获得证据。 |
| J | **技能市场** | 插件中的技能管理页面，用于浏览、安装、启用、归档或删除技能。 |
| J | **计算范式** | 用 Agent、环境和规则把机制实例化，并通过模拟检验。 |
| J | **集成式研究环境（Integrated Research Environment）** | 把文献、假设、实验、仿真、分析和写作放进同一个工作流，而不是分散在多个工具里。 |

### L-M

| 索引 | 术语 | 说明 |
|------|------|------|
| L | **理论范式** | 提出概念、假设、机制和解释框架。 |
| M | **MCP** | Model Context Protocol，让 Claude Code 等编码助手连接外部工具、远程 HTTP/SSE 服务或本地后端。 |
| M | **命令面板（Command Palette）** | VS Code 的快捷命令入口，按 `Ctrl+Shift+P` 或 `Cmd+Shift+P` 打开。 |
| M | **模型名称（Model）** | 实际调用的模型名称。不同服务商的命名不同，建议从服务商控制台复制。 |

### P-R

| 索引 | 术语 | 说明 |
|------|------|------|
| P | **PersonAgent** | AgentSociety² 中默认的人物智能体，用于模拟具有画像、记忆、意图和行动能力的个体。 |
| R | **Run Directory（运行目录）** | 保存一次实验运行产物的目录，通常包含日志、配置快照、结果和回放数据。 |

### S-T

| 索引 | 术语 | 说明 |
|------|------|------|
| S | **Silicon Scientist** | 论文语境里的术语，指协助提出假设、设计实验、分析数据和写作的 Agent。欢迎页中通常称为“研究助手”。 |
| S | **Silicon Subject** | 论文语境里的术语，指作为实验对象被模拟和观察的 Agent。欢迎页中通常称为“被研究对象”。 |
| S | **Skill** | 可安装的能力模块。它告诉 Agent 或编码助手什么时候使用某种能力、读取哪些文件、产生什么输出。 |
| S | **Steps 配置** | 描述实验要执行哪些步骤的配置，常用于定义提问、干预、运行模拟和保存产物。 |
| S | **数据密集型范式** | 通过检索、处理、统计、可视化和模式发现理解大规模数据。 |
| S | **实验配置（Experiment Config）** | 定义实验如何初始化、运行和保存结果的配置文件集合。 |
| T | **Token** | 模型处理文本的计量单位，通常也会影响调用成本。 |

### V-Z

| 索引 | 术语 | 说明 |
|------|------|------|
| V | **VS Code 工作区** | 当前打开的项目目录。插件会在这里读写 `.env`、实验配置、结果、技能和回放数据。 |
| W | **文献库（Literature Library）** | 用于保存论文 PDF、Markdown 笔记、文献索引和研究材料的项目目录。 |
| Y | **研究工作流** | 从文献、假设、实验设计、仿真、分析到写作的一组连续研究步骤。 |
| Z | **智能体（Agent）** | 能根据目标观察环境、规划行动、调用工具并保留状态的计算实体。 |
| Z | **智能体画像（Profile）** | Agent 的身份、背景、目标、偏好或角色设定。 |
| Z | **自定义模块（Custom Module）** | 用户放在项目中的扩展代码，可用于新增环境机制、工具函数或实验逻辑。 |
