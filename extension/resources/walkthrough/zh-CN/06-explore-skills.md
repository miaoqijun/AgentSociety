## 探索技能市场

**技能（Skills）** 是 Agent 和 Claude Code 的能力模块，决定它们能读取什么上下文、调用什么工具、完成什么研究任务。

---

### 🔧 什么是技能？

你可以把技能理解为“研究工具箱”。每安装一个技能，Agent 或 Claude Code 就多一种可复用能力。比如：

- 安装了「学术文献检索」技能 → Agent 能搜索和阅读论文（需先在 `.env` 配置 MCP 网关）
- 安装了"数据分析"技能 → Agent 能运行 Python 脚本生成图表
- 没有安装任何技能 → Agent 只能聊天，不能执行实际操作

### 内置研究技能

| 技能 | 功能 | 什么时候需要 |
|------|------|-------------|
| 📚 学术文献检索 | 搜索学术论文、提取关键信息 | 做文献综述时（需在配置页「高级」填写 MCP 网关与 API Key） |
| 💡 假设生成 | 从文献中提炼可验证的研究假设 | 提出研究假设时 |
| 🔬 实验设计 | 生成实验配置文件、设计研究流程 | 设计实验时 |
| 📊 数据分析 | 运行分析脚本、生成可视化报告 | 分析实验数据时 |
| ✍️ 论文写作 | 辅助撰写论文各章节 | 写论文时 |

---

### 两种技能类型

| 类型 | 安装位置 | 用途 | 管理方式 |
|------|---------|------|---------|
| **Agent 运行时技能** | `custom/skills/` | Agent 在模拟实验运行时使用 | 通过技能市场安装 |
| **Claude Code 技能** | `.claude/skills/` | 默认给 Claude Code 使用，辅助配置、实验检查、代码开发和结果分析 | 通过技能市场安装 |

> 💡 如果你不确定选哪种：**Agent 运行时技能**用于模拟中的 Agent，**Claude Code 技能**用于 IDE 中的默认协作助手。两者互不影响，可以按任务分别安装。

**Agent 运行时**标签页展示模拟运行时可用的技能。你可以扫描工作区技能、导入 Agent 技能、启用或关闭某个技能。

![Agent 技能管理页面示例](../images/agent-skill-management.png)

**Claude 目录**标签页展示当前项目可用的 Claude Code 技能。**复制内置模板到当前项目** 会把插件内置技能同步到 `.claude/skills/`；**导入 Claude 技能** 可添加你自己的技能目录。

![Claude 技能管理页面示例](../images/claude-skill-management.png)

### 如何使用

1. 打开技能市场，浏览可用技能列表
2. 点击技能卡片查看详细介绍
3. 点击 **安装** 将技能添加到项目
4. 已安装的技能会在侧边栏的项目结构中显示

### 技能管理操作

| 操作 | Agent 技能 | Claude 技能 |
|------|-----------|------------|
| 关闭/禁用 | 归档（移到归档目录） | 关闭（不删除文件） |
| 重新启用 | 从归档恢复 | 重新启用 |
| 永久删除 | 从磁盘删除 | 从磁盘删除 |

> 💡 你也可以在 VS Code 设置中配置 `agentSkills.skillSources`，从指定的 GitHub 仓库安装社区开发的技能。
